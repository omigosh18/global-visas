1. Admin Login

POST /api/auth/admin-login

Authenticates an admin user and returns a session cookie.

Request

{
  "email": "shaan.codereve@gmail.com",
  "password": "admin"
}

Response

{
  "id": "ad9ea782-...",
  "email": "shaan.codereve@gmail.com",
  "firstName": "Admin",
  "lastName": "User",
  "role": "admin",
  "emailVerified": true
}
Important: Save the connect.sid cookie from this response. Include it in ALL subsequent requests.

3. Get Single Application

GET /api/admin/applications/:id

Returns a single application by ID with full details.

Headers

Cookie: connect.sid=<session_cookie>

Example

GET /api/admin/applications/d5a977ad-c5ba-4edb-8cc4-2a1e7d4ce07c

4. Update Application Status

PATCH /api/admin/applications/:id/status

Updates the status of an application and optionally adds admin notes.

Headers

Cookie: connect.sid=<session_cookie>

Request

{
  "status": "approved",
  "adminNotes": "Application approved after document verification"
}

Available Status Values

StatusDescription
draftApplication is being filled out by applicant
pendingApplication submitted, awaiting review
under_reviewApplication is being reviewed by admin
approvedApplication has been approved
rejectedApplication has been rejected
additional_info_requiredMore information needed from applicant

5. Delete Application

DELETE /api/admin/applications/:id

Permanently deletes an application and all associated documents.

Headers

Cookie: connect.sid=<session_cookie>

Warning: This action cannot be undone. All documents attached to this application will also be deleted.

n8n Setup Tips

How to Use in n8n

  1. Use the HTTP Request node for all API calls
  2. For the login call, enable "Send Cookies" and "Save Cookies" in the HTTP Request node settings
  3. The session cookie will automatically carry forward to all subsequent requests in the workflow
  4. Always start your workflow with the Admin Login call first
  5. Chain other API calls after the login node

Workflow Example

Auto-approve workflow: Login → Get All Applications → Filter pending ones → Update Status to "under_review" or "approved"

Notification workflow: Login → Get All Applications → Filter new submissions → Send email/Slack notification

Stats reporting: Login → Get Dashboard Stats → Send daily report via email
